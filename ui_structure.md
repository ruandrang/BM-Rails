# 화면 구조 및 디자인 시스템 가이드 (UI Structure)

이 문서는 농구 클럽 매니저 앱에 적용된 Tailwind CSS + DaisyUI 기반의 화면 구조와 디자인 시스템을 설명합니다.

## 1. 디자인 시스템 (Design System)

### 컬러 팔레트 (Color Palette)
- **Primary**: `#FF6B35` (농구공 오렌지) - 주요 액션 버튼, 강조 포인트
- **Secondary**: `#004E89` (코트 블루) - 보조 정보, 헤더 그라데이션
- **Accent**: `#F7B801` (하이라이트 옐로우) - 상태 배지, 경고/강조
- **Neutral**: `#2C2C2C` (다크 그레이) - 텍스트, 그림자
- **Base-100**: `#FFFFFF` (화이트) - 기본 배경색

### 타이포그래피 (Typography)
- **Page Title**: `text-4xl font-bold`
- **Section Title**: `text-2xl font-bold`
- **Card Title**: `text-xl font-bold`
- **Body Text**: `text-base` (사용자 친화적인 가독성 중심)

---

## 2. 공통 레이아웃 (Common Layout)

모든 페이지는 `application.html.erb`에서 정의된 일관된 구조를 따릅니다.

1.  **상단 네비게이션**: 로고 및 사용자 메뉴 (로그아웃, 관리자 링크)
2.  **메인 컨텐츠 영역**: `max-w-7xl mx-auto`를 사용하여 중앙 정렬된 가변 너비 제공
3.  **뒤로가기 버튼**: 페이지 상단 좌측에 항시 배치 (`btn-ghost` 스타일)
4.  **페이지 헤더**: 대제목(`h1`)과 보조 설명(`p`)으로 구성된 영역

---

## 3. 주요 페이지별 구조 (Page Hierarchy)

### 🏀 클럽 메인 (Clubs Index)
- **용도**: 사용자가 참여 중인 클럽 목록 확인
- **구조**: 
    - 상단 '새 클럽 만들기' 액션 버튼
    - 클럽별 카드 그리드 (이름, 아이콘, 멤버수 포함)
    - 빈 상태(Empty State) 디자인 포함

### 🏟 클럽 대시보드 (Clubs Show)
- **용도**: 클럽의 통합 관리 허브
- **구조**:
    - **그라데이션 헤더**: 클럽 아이콘 및 주요 통계 요약
    - **빠른 액션 카드**: 멤버 관리, 경기 기록, 통계, 새 경기 만들기
    - **최근 경기 요약**: 최근 5개 경기 데이터를 담은 소형 테이블
    - **멤버 미리보기**: 상위 8명의 멤버 카드 그리드
    - **데이터 백업**: JSON 내보내기/가져오기 섹션

### 👥 멤버 관리 (Members Index)
- **용도**: 선수 명단 관리 및 순서 변경
- **구조**:
    - **포지션 통계 카드**: G/F/C 및 전체 가용 인원 요약 (Stats 컴포넌트)
    - **CSV 액션**: 내보내기 및 가져오기 폼
    - **카드 그리드**: 멤버별 상세 카드 (이름, 포지션, 나이, 키, 번호)
    - **드롭다운 메뉴**: 카드 내 수정/삭제 기능 통합

### 🏆 경기 기록 (Matches Index & Show)
- **Index**: 
    - 경기 일자별 카드 그리드
    - 진행 상태(완료/진행중) 배지 표시
- **Show (스코어보드)**:
    - 경기 날짜 및 정보 헤더
    - **팀별 카드**: 전적(승/무/패) 및 소속 멤버 명단
    - **결과 기록 섹션**: 게임별 결과(승/패/무)를 입력하는 폼

### 📊 통계 (Statistics Index)
- **용도**: 선수별 퍼포먼스 분석
- **구조**:
    - **통계 요약 카드**: 전체 평균 승률 및 최고 승률 선수(MVP) 표시
    - **데이터 테이블**: 제브라 패턴의 테이블로 승률, 경기수, 승/무/패 상세 표시
    - **정렬 기능**: 이름, 경기수, 승률 기준 필터링

### ⚙️ 관리자 대시보드 (Admin Dashboard)
- **용도**: 전체 플랫폼 데이터 모니터링
- **구조**:
    - **종합 통계**: 사용자, 클럽, 멤버, 경기 수 등의 핵심 지표
    - **데이터 관리 카드**: 각 모델별 상세 목록으로 이동하는 네비게이션 카드

## 4. 주요 컴포넌트 패턴

### 카드 (Cards)
```html
<div class="card bg-base-100 shadow-xl hover:shadow-2xl transition-all">...</div>
```
- 흰색 배경에 그림자 효과, 호버 시 상승 애니메이션 적용

### 배지 (Badges)
- 포지션, 팀 색상, 경기 상태 등을 구분하기 위한 아이콘형 태그

### 폼 (Forms)
- `input-bordered`, `select-bordered`를 사용한 깔끔한 입력창 스타일
- 제출 버튼은 대형(`btn-lg`) `primary` 색상 적용
